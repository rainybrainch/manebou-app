# マネぼう ユーザーフィードバック テンプレート

## 📋 フィードバック方法

マネぼうの改善に向けて、ユーザーからの意見を大切にしています。  
以下のテンプレートを使用して、フィードバックをお送りください。

## ✉️ フィードバック提出方法

### 提出先

| チャネル | 用途 | 回答時間 |
|---------|------|---------|
| **GitHub Discussions** | 機能提案・一般質問 | 1-3営業日 |
| **GitHub Issues** | バグ報告 | 24時間以内 |
| **メール** | 個別相談 | 2-3営業日 |

### 提出前のチェック

- [ ] 既知の課題（KNOWN_ISSUES.md）に記載されていないか確認した
- [ ] 他のユーザーの報告と重複していないか確認した
- [ ] 環境情報（ブラウザ・OS）を確認した

## 📝 フィードバックテンプレート

### 🟢 良かった点

**どの機能が良かったか、理由とともに記載してください。**

```markdown
## 良かった点

- 💚 **[機能名]** – [理由]
  例: ランキング表示がゲーム性を高めていて、生徒のモチベーション向上に役立つ

- 💚 **[機能名]** – [理由]
  例: ダッシュボードの統計情報が見やすく、一目でマネの状況が分かる

*その他あればお書きください*
```

### 🟡 使いづらかった点

**どの部分が改善が必要か、理由とともに記載してください。**

```markdown
## 使いづらかった点

**症状**: [何が使いづらかったか]
**原因の推測**: [なぜだと思うか]
**期待動作**: [どうあるべきか]

例:
- **症状**: マネ付与後、生徒一覧に戻ると更新されていない
- **原因の推測**: ページをリロードしないと反映されていない可能性
- **期待動作**: マネ付与直後に一覧が自動更新されるべき
```

### 🔴 バグ報告

**問題のある動作を詳細に報告してください。**

```markdown
## バグ報告

**【重要】概要**: [簡潔な説明]
例: ログイン後、先生アカウントが学生ダッシュボードに遷移する

**再現手順**:
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]

例:
1. 先生アカウント（teacher@school.jp）でログイン
2. ログインボタンをクリック
3. ダッシュボードが表示される（❌ 先生用ダッシュボードが表示されない）

**期待動作**: [本来の動作]
例: 先生用ダッシュボード（生徒一覧・マネ管理が表示される）が表示されるべき

**実際の動作**: [問題のある動作]
例: 学生用ダッシュボード（残高・マネ履歴が表示される）が表示される

**環境情報**:
- OS: Windows 11
- ブラウザ: Chrome 最新版
- デバイス: PC
- ユーザーロール: teacher
- アプリバージョン: v1.0.0

**スクリーンショット**: [あれば添付]

**Console エラー**: [あれば記載]
例: `Uncaught TypeError: Cannot read property 'role' of undefined`
```

### 💡 欲しい機能

**実装してほしい機能を提案してください。**

```markdown
## 欲しい機能

**機能名**: [提案する機能]
例: メール通知機能

**説明**: [なぜ必要か、どう使うか]
例: マネを付与されたときに、メール通知があると、生徒が即座に気づける

**ユースケース**: [具体的な使用シーン]
例:
- 先生がマネを付与
- 生徒にメール通知される
- 生徒がすぐにアプリを開いて確認

**優先度**: [HIGH / MEDIUM / LOW]
例: HIGH（生徒のエンゲージメント向上に直結）

**実装案**: [可能であれば技術的な提案]
例: SendGrid 統合で実装可能
```

## 📊 フィードバック記入例

### 完全な例

```markdown
# マネぼう フィードバック

## 良かった点

- 💚 **ランキング表示** – メダル（🥇🥈🥉）の表示が視覚的で、ゲーム性が高い
- 💚 **ダッシュボード** – 統計情報が見やすく、マネの状況が一目で分かる
- 💚 **ログイン** – デモユーザーボタンで即座にログインでき、試しやすい

## 使いづらかった点

**症状**: マネ付与後、先生用の生徒一覧ページに戻ると、生徒の残高が更新されていない
**原因の推測**: ページをリロードしないと反映されていない
**期待動作**: マネ付与直後に一覧が自動更新されるべき

## バグ報告

なし（v1.0.0では大きな問題は見当たりませんでした）

## 欲しい機能

**機能名**: メール通知機能

**説明**: マネを付与されたときに、生徒にメール通知があると良い。
生徒が即座に気づけるようになり、学習意欲が高まると予想。

**優先度**: HIGH

## 総合評価

**満足度**: ⭐⭐⭐⭐ 4点

**理由**: 
- UI が直感的で使いやすい
- ゲーム性が高く、生徒のモチベーション向上に役立つ
- パフォーマンスが良好（ページ表示が速い）

**改善への期待**: v1.1.0 でメール通知機能が実装されることを期待しています。
```

## 🔄 フィードバック処理フロー

1. **受け取り** → テンプレートに従ってレビュー
2. **分類** → バグ / 機能提案 / 改善提案に分類
3. **優先度付け** → MoSCoW 方式で優先度を決定
4. **対応** → ROADMAP に組み込みまたは既知の課題に記載
5. **返信** → フィードバック提供者に対応状況を連絡

## 📈 フィードバック統計

- ユーザーロール別（学生 / 先生 / 管理者）
- 対応状況（対応済み / 予定 / 検討中 / 見送り）
