# マネぼう サポートガイド

## 📞 ユーザーサポート体制

このドキュメントは、マネぼうのサポートチーム向けの対応手順書です。

---

## 🎯 サポート対象

### サポート対象外

- **デモユーザーのデータ削除**: デモ環境のため、データ消失は仕様です
- **セキュリティ脆弱性**: 本社セキュリティチームに報告
- **ビジネス相談**: セールスチームに転送

---

## 📋 問い合わせ対応フロー

### ステップ1: 問い合わせ受け取り

**受け取り方法:**
- GitHub Issues
- メール
- チャット

**確認項目:**
- ユーザーロール（学生/先生/管理者）
- 使用環境（PC/スマートフォン）
- ブラウザ（Chrome/Safari/Firefox）
- 発生日時
- エラーメッセージ（あれば）

### ステップ2: 初期対応

```markdown
# サンプル対応テンプレート

ご報告ありがとうございます。

確認させていただきたい項目:
- [ ] ブラウザのキャッシュをクリアしましたか？
- [ ] ページをリロードしてみましたか？
- [ ] 別のブラウザで試しましたか？

上記をお試しいただき、改善しない場合はお知らせください。
```

### ステップ3: トラブルシューティング

**よくある問題と対応**

| 問題 | 対応 |
|------|------|
| ログインできない | localStorage クリア → リロード |
| データが消えた | デモ環境仕様・本番環境で改善予定 |
| ページが遅い | Vercel ステータス確認・キャッシュクリア |
| API エラー | Console 確認・再度試す |

### ステップ4: エスカレーション

**以下の場合は管理者に報告:**
- Console に `Error: ...` が表示される
- 同じ症状の報告が複数ある
- 本番環境に影響がある

## 📝 よくある質問（FAQ）

### Q1: データが消えた

**A:**
- デモ環境では、ブラウザをクリアするとデータが消えます
- 本番環境ではデータベースに保存されます
- 確認方法: localStorage を開く → 該当ページが存在するか確認

### Q2: ログインできない

**A:**
1. メールアドレスが正確か確認
2. ブラウザのキャッシュをクリア
3. 別のブラウザで試す
4. 改善しない場合は管理者に報告

### Q3: スマートフォンで表示がおかしい

**A:**
1. ブラウザをアップデート
2. キャッシュをクリア
3. 画面を回転させて試す
4. 改善しない場合は使用機種とブラウザを報告

### Q4: ランキングが更新されない

**A:**
1. ページをリロード
2. 数秒待機（API応答を待つ）
3. 改善しない場合は Console でエラーを確認
