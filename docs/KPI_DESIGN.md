# マネぼう KPI設計

## 📊 v1.0.x 運用管理指標

マネぼうの成長と健全性を測定するための KPI（主要業績指標）を定義します。

**測定開始日**: 2026-07-08

---

## 📈 KPI 一覧

### 1. ユーザー数関連

#### 🏫 登録学校数

**目的**: サービスの地理的広がりを測定

| 項目 | 値 |
|------|-----|
| **KPI** | 登録学校数 |
| **目標（1ヶ月）** | 3 校 |
| **目標（3ヶ月）** | 10 校 |
| **目標（6ヶ月）** | 20 校 |
| **測定方法** | Admin ダッシュボード → 登録学校一覧 |
| **計測基準** | 実際にマネぼうを導入済みの学校 |
| **確認頻度** | 週 1 回（毎週月曜） |
| **現在状況** | デモ段階・実登録なし |

#### 👨‍🏫 登録先生数

**目的**: 教師側の採用を測定

| 項目 | 値 |
|------|-----|
| **KPI** | 登録先生数 |
| **目標（1ヶ月）** | 5 人 |
| **目標（3ヶ月）** | 20 人 |
| **目標（6ヶ月）** | 50 人 |
| **測定方法** | ユーザーテーブル → role = 'teacher' で COUNT |
| **計測基準** | 実際に登録されたユーザー |
| **確認頻度** | 週 1 回 |
| **実装状況** | ✅ 計測可能（DB 実装後） |

#### 🎓 登録生徒数

**目的**: 生徒側の採用を測定

| 項目 | 値 |
|------|-----|
| **KPI** | 登録生徒数 |
| **目標（1ヶ月）** | 100 人 |
| **目標（3ヶ月）** | 500 人 |
| **目標（6ヶ月）** | 1,000 人 |
| **測定方法** | ユーザーテーブル → role = 'student' で COUNT |
| **計測基準** | 実際に登録されたユーザー |
| **確認頻度** | 週 1 回 |
| **実装状況** | ✅ 計測可能（DB 実装後） |

### 2. アクティビティ関連

#### 📱 月間アクティブユーザー（MAU）

**目的**: 継続利用度を測定

| 項目 | 値 |
|------|-----|
| **KPI** | MAU（月間ログイン者数） |
| **目標** | 登録生徒数の 70% |
| **測定方法** | ログイン日時を記録 → 過去 30 日以内でユニークカウント |
| **計測基準** | 1 回以上ログインしたユーザー |
| **確認頻度** | 週 1 回（毎週月曜・先週分） |
| **実装状況** | ⚠️ 未実装（v1.1.0 で実装予定） |

#### 🎯 週間利用率

**目的**: 週単位での活動度を測定

| 項目 | 値 |
|------|-----|
| **KPI** | 週間利用率（日本時間 月～金） |
| **目標** | 60% 以上 |
| **測定方法** | ログイン数 → 平均生徒数で割る |
| **計測基準** | 教室授業がある平日のみ |
| **確認頻度** | 週 1 回 |
| **実装状況** | ⚠️ 未実装（v1.1.0 で実装予定） |

#### ⏱️ 平均利用時間

**目的**: エンゲージメントを測定

| 項目 | 値 |
|------|-----|
| **KPI** | 1 セッションあたりの平均時間 |
| **目標** | 5 分以上 |
| **測定方法** | セッション開始 → セッション終了で時間計測 |
| **計測基準** | ログイン → ログアウト の時間 |
| **確認頻度** | 週 1 回 |
| **実装状況** | ⚠️ 未実装（v1.1.0 で実装予定） |

| **実装状況** | ✅ 計測可能（GitHub で確認） |

### 5. ビジネス関連

#### 📊 継続率

**目的**: サービス粘着性を測定

| 項目 | 値 |
|------|-----|
| **KPI** | 1 ヶ月継続率（D30） |
| **目標** | 70% 以上 |
| **測定方法** | 導入初月のユーザー → 30 日後のログイン数 |
| **計測基準** | 初回ログイン日 + 30 日以内のログイン |
| **確認頻度** | 月 1 回（月末） |
| **実装状況** | ⚠️ 未実装（ユーザーテーブルで実装可） |

#### 💬 NPS（Net Promoter Score）

**目的**: 推奨度を測定

| 項目 | 値 |
|------|-----|
| **KPI** | NPS スコア |
| **目標** | 50 以上 |
| **測定方法** | 「友人に勧めるか？」を 0-10 で評価 |
| **計測基準** | 9-10 が Promoter、7-8 が Passive、0-6 が Detractor |
| **確認頻度** | 月 1 回 |
| **実装状況** | ⚠️ 未実装（アンケート導入で実装） |

## 📅 測定スケジュール

### 日次
- ページビュー数（トレンド確認）

### 週次（毎週月曜 09:00 UTC）
- ログイン数
- マネ付与回数
- バグ報告数
- アクティブユーザー数

### 月次（毎月 1 日）
- 登録ユーザー数
- 継続率（D7/D30）
- ユーザー満足度アンケート
- NPS アンケート

### 四半期（3 ヶ月ごと）
- ユーザー成長分析
- コホート分析
- チャーン分析
- ビジネス目標の見直し
